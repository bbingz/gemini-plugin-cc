# Cross-Plugin Alignment Reports

本目录记录 Gemini 插件（本仓库）与三个姐妹插件的对齐状况。

## 方法论

每份对齐报告遵循同一结构：

1. **我的观察** — 版本、命令面、CLI 路径（纯事实）
2. **对齐度对照表** — 把 baseline 的每一项能力逐条打勾/打叉
3. **我看到的独有设计** — 对方做得比我好或不同的地方，公正记录
4. **我视角的差距** — 按优先级列出，每条都给出定位到行号的证据
5. **我看不到的地方** — 我的盲区（设计意图、v0.x 计划、外部 CLI 行为差异），明确标注**不要**当成 TODO，而是**请你来澄清或补充**
6. **迁移建议** — 如果我自己的某段实现可复用，我指出文件和行号

## 三条原则

- **证据优先**：每条结论都给到 `file:line`，方便复核
- **承认盲区**：我不替你们做决策。我说"我看不见 X，原因是 Y"，由你们判断是有意省略还是待做
- **单向流动**：这些文档只写给你们看，**我不会主动去改你们的代码**。如果你们想要我动手，请把请求发回 Gemini 插件这边

## 关于"A 卷"术语

上游对话里 bbing 用"A 卷"指代**主力/高档模型是否被真正服务**（vs. 静默降级到低档模型）。在 Gemini 插件里这由 `stats.models` 字段的解析承担（参见 `baseline.md` §6.3）。基线文档会把术语统一到 **"主力模型使用验证 (primary-model attestation)"**，避免歧义。
